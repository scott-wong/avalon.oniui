progressbar

进度条 progressbar组件，可以通过接口控制或者随机模拟进度条效果，支持条形，圆形，倒计时等效果功能
使用说明
名字默认值说明
配置参数
toggle true 组件是否显示，可以通过设置为false来隐藏组件
value false 当前进度值 0 - 100 or false
label true 是否在进度条上显示进度数字提示
simulate false 是否模拟进度条效果，默认为否，模拟的时候需要调用触发告知完成，模拟会采用模拟函数及算子进行模拟，取值为int表示动画效果间隔ms数
indeterminate false 是否不确定当前进度，现在loading效果
countDown false 倒计时
inTwo false 是否显示左右两段
circle false 圆形
getStyleFromSkin true 是否从皮肤的css里面计算获取圆形进度条样式，默认为true，设置为true的时候，将忽略下面所有circle*样式设置
circleColor "#ffffff" 圆形填充色彩，可以配制为从皮肤中提取，只在初始化的时候提取
circleBorderColor "#dedede" 圆形边框颜色，，可以配制为从皮肤中提取，只在初始化的时候提取
circleBarColor "#619FE8" 圆形进度条边框颜色，可以配制为从皮肤中提取，只在初始化的时候提取
circleRadius 0 圆形的半径，可以配制为从皮肤中提取，只在初始化的时候提取
circleBorderWidth 0 圆形的边框宽度，可以配制为从皮肤中提取，只在初始化的时候提取
success false 是否完成，进度为100时或者外部将success置为true，用于打断模拟效果
onInit回调函数onInit(vmodel, options, vmodels) 完成初始化之后的回调,call as element's method
simulater配置接口simulater(value, vmodel) 模拟进度进行效果函数，参数为当前进度和vmodel，默认return value + 5 * Math.random() >> 0
getTemplate配置接口getTemplate(tmp, opts) 用于修改模板的接口，默认不做修改
getTemplate配置接口getTemplate(tpl, opts) 定制修改模板接口
onChange回调函数onChange(value) value发生变化回调，this指向vmodel
onComplete回调函数onComplete() 完成回调，默认空函数，this指向vmodel
labelShower配置接口labelShower(value, isContainerLabel) 用于格式化进度条上label显示文字，默认value为false显示“loading…”，完成显示“complete!”，失败显示“failed!”，第二个参数是是否是居中显示的label，两段显示的时候，默认将这个label内容置空，只显示两边的label,this指向vmodel
start接口调用start() 开始进度推进，该接口适用于模拟进度条
end接口调用end(value) 结束进度推进，该接口适用于模拟进度条，value为100表示结束，failed表示失败，undefine等于pause，其他则终止于value，并在label上显示
reset接口调用reset(value) 重置设置项，参数可选，为需要重设的值
progress接口调用progress(value) 设置value值，其实也可以直接设置vmodel.value
_draw接口调用_draw 动态切换皮肤之后，如果需要更新提取圆形进度条的样式，可以调用这个方法
