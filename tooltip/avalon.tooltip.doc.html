tooltip

气泡提示组件 tooltip组件，给一个元素或者给元素里满足配置条件的系列元素添加一个富UI及交互的气泡提示框
使用说明
名字默认值说明
配置参数
toggle false 组件是否显示，可以通过设置为false来隐藏组件
collision "none" 溢出检测，当被定位元素在某些方向上溢出窗口，则移动它到另一个位置。与 my 和 at 选项相似，该选项会接受一个单一的值或一对 horizontal/vertical 值。例如："flip"、"fit"、"fit flip"、"fit none"。
"flip"：翻转元素到目标的相对一边，再次运行 collision 检测一遍查看元素是否适合。无论哪一边允许更多的元素可见，则使用那一边。
"fit"：把元素从窗口的边缘移开。
"flipfit"：首先应用 flip 逻辑，把元素放置在允许更多元素可见的那一边。然后应用 fit 逻辑，确保尽可能多的元素可见。
"none": 不检测
event "mouseenter" 显示tooltip的事件，默认hover的时候显示tooltip，为false的时候就不绑定事件，如果后面设置了自动隐藏，则mouseenter对应的是mouseleave,focus对应的是blur，进行自动隐藏事件侦听，使用代理的时候，目测不支持focus,blur，event可以配置为空，则不会添加事件侦听
width "auto" tip宽度，默认是auto
height "auto" tip高度，默认是auto
arrow true 是否显示尖角图标，默认为true
autohide true 元素hoverout之后，是否自动隐藏tooltip，默认true
delegate false 元素是否只作为一个代理元素，这样适合对元素内多个子元素进行tooltip绑定
disabled false 禁用
track false tooltip是否跟随鼠标，默认否
animated true 是否开启显示隐藏切换动画效果
position "rt" tooltip相对于element的位置，like: "rt,rb,rc..."
positionMy false tooltip元素的定位点，like: "left top+11"
positionAt false element元素的定位点，like: "left top+11",positionAt && positionMy时候忽略position设置
hiddenDelay 64 tooltip自动隐藏时间，单位ms
onInit回调函数onInit(vmodel, options, vmodels) 完成初始化之后的回调,call as element's method
contentGetter配置接口contentGetter() 获取内容接口，讲srcElement作为参数传递过来，默认是返回a标签的title，如果该函数返回为空，那么则不会显示tooltip
_animateArrMaker配置接口_animateArrMaker(from, to) 不支持css3动画效果步长生成器函数，返回一个数组，类似[0,xx,xx,xx,50]
getTemplate配置接口getTemplate(tpl, opts) 定制修改模板接口
show接口调用show(elem) 不建议使用这个方法，请使用showBy({target: ele})显示tooltip，相对于elem定位，elem为元素或者event事件对象，如果elem为空，则采用之前缓存的对象，两者都为空，则只展示，不改变位置
hide接口调用hide($event) 隐藏tooltip，参数是$event，可缺省
showBy接口调用showBy($event) 参数满足 {target: elem}这样，或者是一个elem元素亦可，tooltip会按照elem定位，并作为参数传递给contentGetter

组件在不跟随鼠标移动时候，会自动将箭头对齐元素中间的位置，当tooltip没有元素高或者宽时，会自动调整tip显示在元素高度或者宽度居中的地方

tooltip 组件有一个重要的contentGetter方法,见源码：

   "contentGetter": function(elem) {
        if(elem.tagName.toLowerCase() != "a") return
        return elem.title
    }

该函数有无返回值，将决定是否展示tooltip

例子：

  1. input元素不绑定事件，人工触发
  2. input元素hover显示tip-父容器做代理
  3. 不使用代理，各个方向绑定，改变位置，点击展示
  4. 箭头位置自动调整，点击展示
  5. input元素获取焦点显示tip，不展示箭头
  6. 鼠标跟随
  7. input元素获取焦点显示tip
